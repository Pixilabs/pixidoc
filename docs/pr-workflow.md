# Pull request workflow

## Tests

Vérifier que tous les tests ont bien été implémentés.

## Pixidoc

Avant de fermer une pull request, vous devez vérifier que la documentation ait bien été mise à jour.

## Composants

Si la PR ajoute de nouveaux composants, il faut s'assurer que les screenshots de ces composants sont disponibles sur le drive dans `Pixilabs > Ressources > Screenshots > Briques`.

Il faut également s'assurer que les nouveaux composants sont ajoutés à la briquothèque. (Faire une tâche Trello dans le tableau Marketing)
